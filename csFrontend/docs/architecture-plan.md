# 프론트 리디자인 설계 요약 (4서비스 통합 기준)

## 1) 목표
- 기존 서비스명 노출 UI -> 업무 중심 메뉴 UI로 전환
- Admin + Master 통합(AdminCore) 구조를 프론트 라우팅과 권한정책에 반영
- 백엔드 미완성 상태에서도 프론트 진행 가능하도록 mock store + API 계약 상수 분리

## 2) 폴더 구조 전략
- `src/app/*`: 라우트 페이지 (얇게 유지)
- `src/features/*`: 화면 단위 도메인 로직/컴포넌트
- `src/shared/*`: 공통 타입, 레이아웃, 상태, 유틸, API 계약
- `public/assets/*`: 배경 이미지 등 정적 자산
- `docs/*`: 설계도/흐름도 산출물

## 3) 핵심 규칙 반영
- 총 운영 인원 30명 제한 (대기+진료중+예약+응급)
- 응급 카운터 0~10
- 접수/예약 대시보드 조회 전용
- 주민번호 기본 마스킹 / ADMIN,SYS만 전체보기 + 감사로그 기록
- 검사/영상 복수선택 + 그룹별 '없음' 상호배타
- 최종오더 NONE 단독 규칙
- 결제 규칙: 약제만 처방인 경우 결제 후 완료 처리, 수술/입원 포함이면 완료 자동 미적용

## 4) 실제 백엔드 연동 시 권장 순서
1. 로그인/JWT/권한 가드 연동 (IAM)
2. AdminCore 대시보드/접수/응급 API 연동
3. Clinical SOAP/Orders/FinalOrders 연동
4. Billing 자동산출/결제 연동
5. Master 직원/카탈로그 연동
